# v0.5.0

Released: 2026-03-14

## Highlights

- Phase 5 dashboard work is now implemented end to end:
  - speed test widget and backend
  - Wake-on-LAN widget and restricted backend endpoint
  - live firewall log streaming
  - drag-and-drop layout persistence
- The dashboard backend is now threaded, which removes the worst UI stalls when long-running endpoints are active.
- Monitoring integrations are materially more usable:
  - Fail2ban status works from the dashboard
  - Technitium DNS statistics use the current API field layout
  - interface widgets map correctly to logical roles
  - service status parsing is more reliable

## Added

- Caddy diagnostics widget
- Technitium log page
- Fail2ban status page
- Quick Links copy action support for targets like SSH commands
- Netdata UI support through the `pkgs.netdataCloud` package path in host deployments

## Fixed

- Dashboard CPU usage reporting no longer gets stuck at `0.0%`
- Dashboard CPU reporting is smoothed to better match host tools like `btop`
- Browser cache and saved layout issues are documented so newly added widgets are easier to surface
- Firewall flowtable/offload visibility works again when the host runtime unit creates the nft flowtable successfully
- Caddy startup no longer fails on agenix secret permissions during `ExecStartPre`

## Operational Notes

- Consumers of this flake should deploy new dashboard code by pushing commits and updating their flake lock, not by switching to a temporary `path:` input in shared host configs.
- Technitium cache metrics currently come from `totalCached` and `cachedEntries`, not `totalCachedQueries`.
- Netdata deployments that want the browser UI should use `pkgs.netdataCloud`; `pkgs.netdata` may build without bundled dashboard assets in current nixpkgs.

## Known Limitations

- Phase 3 service control is still deferred.
- Phase 4 nftables hit counters and richer firewall breakdowns are still partial.
- DHCP multi-scope/multi-interface presentation work is in progress locally and is not part of this release.
