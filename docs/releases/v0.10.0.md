# v0.10.0

Released: 2026-04-24

## Highlights

- Kea HA deployments can now advertise a peer-reachable local control endpoint instead of always hard-coding `127.0.0.1`.
- HA-enabled Kea now loads the lease command hook automatically, which restores the command surface required for lease synchronization (`lease4-get-page`, etc.).
- DHCPv4 now exposes an explicit `outbound-interface` option so consumers can select `use-routing` when validating HA/VRRP broadcast behavior.

## Added

- `services.router-kea.dhcp4.ha.localAddress` for declaring the address this node should publish for its own Kea HA listener.
- `services.router-kea.dhcp4.outboundInterface` with supported values:
  - `same-as-inbound`
  - `use-routing`

## Fixed

- Kea HA configurations no longer implicitly self-advertise `http://127.0.0.1:8000/` in multi-node deployments.
- HA-enabled Kea now loads `libdhcp_lease_cmds.so` alongside `libdhcp_ha.so`, avoiding lease-sync failures caused by missing lease command APIs.

## Operational Notes

- Consumers should set `services.router-kea.dhcp4.ha.localAddress` to a per-node address reachable by the HA peer. In VRRP topologies this should generally be the node's stable non-VIP service address, not the shared virtual IP.
- `outboundInterface = "use-routing"` is intended as an explicit control for topologies where Linux socket delivery semantics need to be validated or worked around; existing consumers can keep the default `same-as-inbound`.
